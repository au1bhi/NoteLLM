# NoteLLM 毕业设计目标

## 项目定位

NoteLLM 是一个面向个人学习与研究的文档问答系统，参考 Google NotebookLM 的核心体验：用户将资料放入笔记本，在限定资料范围内提问，并获得可回溯来源的回答。

本项目是毕业设计原型。目标是证明完整、可信且可评估的 RAG 工作流，不是复刻工业级产品。

## MVP 用户闭环

1. 用户登录并创建笔记本。
2. 上传 PDF、TXT 或 Markdown 文档。
3. 系统解析、分块、向量化并建立索引。
4. 用户提问，系统检索当前笔记本资料并流式回答。
5. 每个回答展示文档名、页码（适用时）和原文片段引用。
6. 用户可重新查看笔记本、来源和对话记录。

## 验收标准

- 笔记本、来源、对话均按用户隔离。
- 回答基于检索到的来源；无法支持时明确说明资料不足。
- 引用可定位至来源和相应文本片段或页码。
- 关键流程具备自动化测试，并可用 Docker Compose 本地演示。
- 使用固定评测集报告检索质量、引用正确率、回答忠实度和响应时间。

## 明确不做

- 多人实时协作、企业级权限、音频概览、移动端 App。
- 网页爬取、OCR、复杂表格/图片理解与全格式文档支持。
- 微服务、消息队列、多模型容灾、超大规模并发和生产级运维。

## 约束与原则

- 每项工作必须服务于可演示的 MVP；先完成端到端流程，再扩展功能。
- 只使用一个可配置的聊天模型和嵌入模型；密钥和提示词仅存在后端。
- 首选 PostgreSQL + pgvector 与本地 Docker Volume，避免引入独立基础设施。
- 用户上传内容是不可信输入，不能覆盖系统指令；删除来源时应清理派生数据。
