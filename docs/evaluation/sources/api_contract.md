# API 契约说明

笔记本列表和详情接口只返回当前用户拥有的对象。来源上传接口接受 multipart 文件，并在同步处理结束后返回来源状态。前端不得假设每个来源都成功：failed 状态需要显示后端提供的错误信息，并允许在配置恢复后重试。

检索接口的 query 必须非空，limit 的范围为 1 到 10，默认是 5。返回结果包括分块 ID、来源显示名、内容、可选页码和相似度。相似度用于排序参考，不应被 UI 解释为模型回答正确的概率。

问答接口用 Server-Sent Events 依次发送 answer delta、citations 和 done 事件。断开连接或供应商失败时，客户端应显示可恢复的错误状态。刷新工作区时，前端应从持久化的 conversation 和 message 数据恢复历史，而不是仅依赖内存状态。
