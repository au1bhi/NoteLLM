# RAG 工作流说明

NoteLLM 的一次问答从资料上传开始。系统只接受 UTF-8 TXT、Markdown 和 PDF。文本资料会保留原始文件名；PDF 会按页提取文本，因此引用可以显示页码。

处理流程依次为：提取文本、按 1,000 个字符分块、相邻分块保留 150 个字符重叠、调用嵌入模型并写入 pgvector。来源状态依次可能为 pending、processing、ready 或 failed。只有状态为 ready 且已有向量的分块才可参与检索。

用户提问时，系统将问题嵌入为向量，并在同一笔记本的 ready 来源中按余弦距离检索前 5 个分块。聊天模型只能依据这些分块生成答案，并返回本次候选分块的 ID。后端会移除不属于候选集的引用；若没有有效引用，回答必须是“资料不足，无法根据当前笔记本中的来源可靠回答。”

来源删除时，数据库会级联删除其分块和向量；本地上传文件也会被删除。不同用户的笔记本查询始终要经过 owner_id 校验，不能依赖客户端传入的笔记本 ID。
