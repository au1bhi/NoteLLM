# NoteLLM 答辩演示资料

NoteLLM 是一个面向个人学习和研究的文档问答毕业设计原型。用户在一个笔记本中上传资料，系统将资料解析为文本分块、生成向量并写入 pgvector。提问时，系统只在当前笔记本的 ready 来源中检索相关分块。

系统支持 UTF-8 TXT、Markdown 和 PDF。文本按 1,000 个字符切分，相邻分块保留 150 个字符重叠。PDF 提取时保留页码；TXT 和 Markdown 没有页码，但回答仍会展示来源文件名和稳定摘录。

答案必须使用本轮检索到的来源。聊天模型只能返回候选 chunk ID，后端会拒绝未知 ID。若没有能支持回答的有效引用，系统应回复“资料不足，无法根据当前笔记本中的来源可靠回答。”这使用户能够区分有资料支持的结论与系统无法验证的问题。

数据按用户隔离。用户只能访问自己的笔记本、来源和会话；访问他人对象时服务返回 404。删除来源会同步删除上传文件、文本分块和向量，避免旧资料继续参与检索。
